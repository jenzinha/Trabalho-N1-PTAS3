API - USER

Tecnologias usadas

routes.post('/user', userController.createUser);

Router: Esse objeto é usado para definir rotas e manipular as solicitações HTTP para essass rotas

Post: É usado para enviar dados para o servidor, muitas vezes para criar novas rotas

userController: Função responsavel por processar a solicitação e realizar as ações necessárias

Essa função é responsavel pela criação de usuario no banco de dados. Usado para enviar dados confidenciais.

routes.get('/users', userController.findUsers);

Get: Usado para recuperar uma informação. Verbo padrão para enviar dados quando submetemos um formulario HTTP

Basicamente o get é usado para enviar as informações menos sensiveis

routes.delete('/user/:id', userController.deleteUser);

Delete: Usado para deletar componentes

deleteUser: pega a função do userController, essa funçao especifica é responsavel por deletar

routes.put('/user/:id', userController.updateUser);

Put: É usado para atualizar ou modificar recursos no servidor.

Ele basicamente atualiza e faz modificações em informações ja criadas.

routes.post('/user/authenticated', userController.authenticatedUser);

POST: É usado para atualizar ou modificar recursos no servidor.

'/user/authenticated': A rota será acionada quando uma solicitação POST for feita para a URL com o caminho "/user/authenticated".

userController.authenticatedUser: Isso se refere à uma função controlada que será executada quando a rota dor acessada.

A função controladora é resposavel por lidar com a logica da rota que se trata da autenticação do user

Express

Express: Usamos o express que é uma ferramenta para desenvolvedores que desejam criar aplicativos web e APIs de forma rápida e eficiente.

Alguns dos seus principais propositos são:

  • Desenvolvimento de aplicativos web
  • Gerenciamento de rotas
  • Middleware
  • -_Integração com bancos de dados_-

Postgress

Postgress: Utilizamos o postgres que é um banco de dados relacional.

Ele é usado para armazenar, gerenciar e recuperar dados.

Algumas das principais finalidades dele:

  • -_Armazenamento de dados_-
  • Análise de dados
  • Segurança e conformidade

Nodemon

Nodemon: Ele é uma biblioteca que ajuda no desenvolvimento de sistemas que usam o node.js reiniciando automaticamente o servidor.

Algumas das suas principais funções são:

  • Reinicialização automática do servidor
  • Recarregamento automático de aplicativos
  • Facilita o desenvolvimento em tempo real

SQL-Client

SQL-Client: É uma ferramenta que permite que o client se conecte a um banco de dados e interaja com ele

Sua principal finalidade é facilitar a administração, consultoria e manipulação dos dados armazenados no banco

Algumas de suas principais caracteristicas são:

  • -_Consulta de dados_-
  • -_Inserção, atualização e exclusão de dados_-
  • -_Gerenciamento de bancos de dados_-
